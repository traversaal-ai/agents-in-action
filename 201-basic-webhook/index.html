    üîÄ Workflow: Basic Webhook

    This blog contains the Basic Webhook workflow for n8n. It shows the simplest ‚ÄúHTTP in ‚Üí AI out‚Äù pattern using a Webhook, AI Agent, OpenRouter, and Memory.

    ‚ú® Overview

    Send a POST request to an n8n Webhook, pass a prompt, and get a short AI answer back. Best tested with the provided Google Colab notebook. You can also try with curl or Postman.

    üöÄ Key Features

    • üß† AI Agent ‚Äî enforces simple, consistent response rules.

    üîÑ How It Works

    graph LR
  A["üåê Webhook (POST)"] --> B["üß† AI Agent"]
  B --> C["ü§ñ OpenRouter Chat Model"]
  B <--> D["üóÇÔ∏è Memory (sessionKey = username)"]
  B --> E["‚Ü©Ô∏è Respond to Webhook"]
    1. Client sends POST JSON ‚Üí Webhook.
    2. AI Agent reads the prompt and rules.
    3. OpenRouter generates a short reply.
    4. Memory uses username to keep per-user context.
    5. The workflow responds to the HTTP request.

    üõÇ Inputs (JSON Body)

    Example

  "username": "demo-user-1"

    üß∞ Troubleshooting

    • 404 / 405 ‚Üí Workflow not Active, wrong method (must be POST), or wrong URL.
    • 401 / 403 ‚Üí Check n8n auth/network; verify your Production URL.
    • Empty/odd response ‚Üí Ensure JSON body has query and username, and header is Content-Type: application/json.
    • Memory not sticking ‚Üí Use a stable username (same value across requests).
